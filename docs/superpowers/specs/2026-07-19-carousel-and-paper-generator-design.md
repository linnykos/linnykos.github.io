# Design — Photo carousel + paper generator

Date: 2026-07-19
Author: Kevin Z. Lin (with Claude)
Status: Approved (design Q&A answered)

Two independent improvements to the Lin Lab website (static GitHub Pages, no
deploy-time build). They share no code and can be reviewed separately.

---

## Feature 1 — Photo carousel on `people.html`

### Goal
Let visitors tap/click through the lab photos (`images/lab-photo-2024.jpg`,
`lab-photo-2025.png`, `lab-photo-2026.png`) instead of showing two stacked
static images. Must work in a normal web browser with no new libraries.

### Requirements
- Newest first: 2026 → 2025 → 2024.
- **Uniform footprint for photos of differing shapes.** The three photos have
  different aspect ratios (2024 ≈ 1.91:1, 2025 ≈ 1.94:1, 2026 ≈ 1.50:1). Solved
  with a fixed-size stage + `object-fit: contain` over a **white** background:
  each photo scales to fit the identical box and any leftover space is filled
  white (letterbox / pillarbox). No photo is ever cropped. This is the answer to
  "add white to the height/width if needed" — yes, it's possible, in pure CSS.
- Handwritten captions, mapped by the last digit of the photo year:
  - 2026 → "Lab photo, taken the 2025-26 academic year."
  - 2025 → "Lab photo, taken the 2024-25 academic year."
  - 2024 → "Lab photo, taken the 2023-24 academic year."

### Design
Replace the current `<!-- New Lab Photos Section -->` block in `people.html`
(the two hard-coded `<img>` elements) with one self-contained carousel:

- **Stage**: `max-width: 500px`, `aspect-ratio: 5 / 3`, white background, one
  `<img>` per slide stacked absolutely; only the `.active` slide is shown.
- **Controls**: ‹ / › buttons overlaid left/right; a row of dot indicators; a
  caption `<p>` under the stage. Buttons and dots are real `<button>`s with
  `aria-label`s.
- **Behavior**: click ‹/› or a dot to change slide; wraps around; ArrowLeft/
  ArrowRight change slides when the carousel has focus.
- **Responsive**: on phones the stage goes full-width and keeps its aspect box.
- CSS + JS live inline in `people.html`, scoped under `#labCarousel` /
  `.carousel*` class names so nothing else on the site is affected. jQuery
  (already loaded) is left untouched; the carousel uses plain vanilla JS.

### Out of scope
Autoplay, swipe/touch-drag gestures, captions auto-generated from filenames.

## Feature 2 — Paper entry generator (`index.html`)

### Goal
Stop hand-writing ~28 lines of HTML per paper. Kevin edits one easy data file
and regenerates the papers region of `index.html`.

### Architecture — local generator (chosen over client-side fetch)
`papers.yml`  →  `python3 generate.py`  →  rewrites `index.html` papers region.

- The deployed HTML stays **fully static and byte-identical in shape to today**
  (good for Google Scholar / SEO; `file://` double-click preview still works).
- The generator rewrites **only** the region between two markers it inserts:
  `<!-- PAPERS:START (generated by generate.py — do not edit by hand) -->` …
  `<!-- PAPERS:END -->`. Everything else in `index.html` (About, Articles,
  banner, footer) is left untouched.
- Format: **YAML**, parsed with PyYAML. One-time `pip3 install pyyaml`
  (machine is Python 3.7, no stdlib TOML). `generate.py` prints the exact
  install command if PyYAML is missing.

### Data model (`papers.yml`)
A top-level list. The generator emits the three section headers
(Preprints/In Preparation, Highlighted published papers, Other published papers)
and, under Preprints, the "(Bold denotes…)" note — these are fixed strings keyed
off `section`. Papers render in file order within each section.

Per-paper fields:

| field      | required | meaning |
|------------|----------|---------|
| `id`       | yes | short label; becomes the `<!-- id -->` comment. |
| `section`  | yes | `preprints` \| `highlighted` \| `other`. |
| `image`    | yes | thumbnail path, e.g. `images/plot-scope.png`. |
| `alt`      | no  | img alt text (defaults to `"<title> figure"`). |
| `title`    | yes | short bold `research_title`. |
| `abstract` | no  | italic abstract paragraph; omit for no abstract. |
| `authors`  | yes | list of `{name, link?, emph?}`. `emph: true` = bold lab member; `link: firstname_lastname` = homepage via `script.js`. Equal-contribution `*` is just part of `name`. |
| `note`     | no  | free text after the author list, e.g. `(*Equal contributions)`. |
| `pubtitle` | yes | full citation title (often longer than `title`). |
| `venue`    | yes | **verbatim** citation line as HTML — journal name (linked/`<i>` as needed), volume, year, trailing colon. Free-form so all the per-journal variation is preserved exactly. |
| `links`    | no  | list of `{label, url}`, each rendered as `(<a href=url target=_blank>label</a>)`. |
| `extra`    | no  | free HTML appended after the links (e.g. NetworkSoSD's "Data (4.15 Gb…)" line). |

Rendered block matches the current structure exactly:

```
<div class="block">
  <div class="abstract">
  <img class="project_img" src="{image}" alt="{alt}">
  <div class="right">
  <div class="research_title">{title}</div>
  <p>{abstract}</p>            (only if abstract present)
  </div>
  </div>
  <div class="publication">
    {authors, joined ", " with "and" before the last, terminal "."} {note}
  <br>
  {pubtitle}<br>
  {venue}
  {links}
  {extra}
  </div>
</div>
```

### Migration + verification
1. Author `papers.yml` for all 25 existing paper blocks.
2. Insert the markers into `index.html`.
3. Run `generate.py`; compare the regenerated region against the original with a
   whitespace-normalizing diff (collapse runs of whitespace / inter-tag
   whitespace, then compare). Target: **no meaningful differences** — any
   residual is cosmetic whitespace only. Iterate `papers.yml`/`generate.py`
   until clean.

### Out of scope
The "Articles" section, the About blurb, and the other three HTML pages remain
hand-edited. The generator only manages the three paper sections.

### Files added
- `papers.yml` — the data.
- `generate.py` — the generator (stdlib + PyYAML).
- `README` note in `generate.py`'s header comment documenting the 2-step
  workflow (edit yml → run script → push).
