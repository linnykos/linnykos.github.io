# How to update this website

A cheat-sheet for future me. This is the personal + lab site served at
<https://linnykos.github.io/>.

## The 10-second mental model

- It's **plain static HTML** — no build system, no framework, no database.
- Hosting is **GitHub Pages**. **Pushing to the default branch = deploying.**
  Wait ~1 minute after pushing, then hard-refresh the live site.
- To **preview before pushing**, just double-click the `.html` file to open it
  in your browser. (The carousel and everything else work from a local file.)
- The one exception to "just edit HTML" is the **publication list**, which is
  now generated from `papers.yml` — see below.

## Files at a glance

| File | What it is |
|---|---|
| `index.html` | Landing page: About + **publications** (generated) + Articles |
| `people.html` | Lab photo carousel, current members, alumni |
| `resources.html` | Application materials, teaching, job links |
| `funding.html` | Grants and awards |
| `papers.yml` | **The publication data you edit** (see below) |
| `generate.py` | Turns `papers.yml` into the publications in `index.html` |
| `script.js` | Maps `data-link="firstname_lastname"` → each person's homepage URL |
| `css/style.css` | All the styling |
| `images/` | `plot-*.png` = paper thumbnails; `lab-*.png` = member headshots; `lab-photo-*` = group photos |
| `papers/` | Hosted PDFs (preprints, CV) |

## Task 1 — Add or update a paper  ⭐ (the common one)

**You no longer hand-edit the HTML.** You edit `papers.yml` and run a script.

1. Add the thumbnail image to `images/` (e.g. `images/plot-mymethod.png`).
2. Open `papers.yml`, copy an existing entry, and edit the fields (template
   below). Put it in the right `section` and in the order you want it to appear.
3. Run the generator from the repo folder:

   ```
   python3 generate.py
   ```

   (First time on a new computer only: `pip3 install pyyaml`. No conda
   environment is needed — PyYAML is the only dependency.)
4. Open `index.html` in a browser to eyeball it, then commit & push.

### Minimal entry (no abstract, one link)

```yaml
- id: mymethod
  section: preprints          # preprints | highlighted | other
  image: images/plot-mymethod.png
  title: "MyMethod: a short bold title"
  authors:
    - {name: "A. Student", link: firstname_lastname, emph: true}
    - {name: "K. Z. Lin", emph: true}
  pubtitle: "The full paper title as it appears in the citation"
  venue: "2026:"
  links:
    - {label: "biorxiv", url: "https://..."}
```

### Full entry (all the knobs)

```yaml
- id: mymethod
  section: highlighted
  image: images/plot-mymethod.png
  alt: MyMethod figure               # optional; image alt text
  title: "MyMethod: short bold title shown above the abstract"
  abstract: >                        # optional; omit the key for no abstract
    Write the abstract here. It can span
    multiple lines; they get joined together.
  authors:
    - {name: "A. Student", link: firstname_lastname, emph: true}
    - {name: "J. Coauthor", link: jane_coauthor}
    - {name: "P. Plainname"}
    - {name: "K. Z. Lin", emph: true}
  note: "(*Equal contributions)"     # optional; text after the author list
  pubtitle: "The full citation title (often longer than `title`)"
  venue: '<a href="https://journal.example/" target="_blank">Journal Name</a> 12.3 (2026):'
  links:
    - {label: "link", url: "https://..."}
    - {label: "code: method", url: "https://github.com/..."}
  extra: '<br>Data: (<a href="..." target="_blank">download</a>)'  # optional; raw HTML after links
```

### Field rules (the ones that trip me up)

- **`section`** must be exactly `preprints`, `highlighted`, or `other`.
- **Author flags:**
  - `emph: true` → **bold** (denotes a Lin Lab member).
  - `link: firstname_lastname` → makes the name a link to their homepage.
    The key must **exactly match** an entry in `script.js` (see Task 3), or the
    link silently breaks. No `link` = plain-text name.
- **`venue`** is free-form HTML on purpose — paste the journal/volume/year line
  however it should look (including `<i>...</i>` or a linked journal name). Put
  the trailing `:` here; the `(links)` are added automatically after it.
- **`links`** each render as `(label)`. For the rare case of two links inside
  one set of parentheses, use `- {raw: "(<a ...>git 1</a>, <a ...>git 2</a>)"}`.
- The generator always writes a consistent "…, and LastAuthor." — that's
  intentional, don't fight it.

### ⚠️ Do not hand-edit the generated block

In `index.html` there is a region fenced by:

```
<!-- PAPERS:START (generated by generate.py - do not edit by hand) -->
... generated papers ...
<!-- PAPERS:END -->
```

Anything you type between those markers is **overwritten** next time you run
`generate.py`. Always edit `papers.yml` instead.

## Task 2 — Add a lab photo to the carousel (`people.html`)

The carousel lives in `people.html` under `<div id="labCarousel">`. To add a
new year's group photo:

1. Drop the image in `images/` (e.g. `images/lab-photo-2027.png`).
2. In `people.html`, inside `<div class="carousel-stage">`, add a new slide.
   To make it the first one shown, put it right after the "prev" button and
   remove `active` from the old first slide, adding it to the new one:

   ```html
   <div class="carousel-slide active">
     <img src="images/lab-photo-2027.png" alt="Lab photo, 2026-27 academic year">
   </div>
   ```
3. Add one more dot in `<div class="carousel-dots">`:

   ```html
   <button type="button" class="carousel-dot" aria-label="Photo 4 (2026-27)"></button>
   ```
   (Only one dot should have `class="carousel-dot active"` — the first slide's.)
4. In the `<script>` just below, add the caption to the `captions` array **in
   the same order as the slides**:

   ```js
   var captions = [
     'Lab photo, taken the 2026-27 academic year.',
     'Lab photo, taken the 2025-26 academic year.',
     ...
   ];
   ```

Photos of any shape are fine — they're auto-fit into a fixed box with white
padding, so they won't be cropped or stretched.

## Task 3 — Add / fix a person's homepage link

Homepage URLs live in `script.js` in the `websiteLinks` dictionary, keyed by
`firstname_lastname`.

1. Add an entry, e.g. `"jane_coauthor": "https://jane.example.com",`.
2. Reference it anywhere in HTML with:

   ```html
   <a class="link" href="" target="_blank" data-link="jane_coauthor">J. Coauthor</a>
   ```
   or, in `papers.yml`, with `link: jane_coauthor`.

The key must match exactly on both sides or the link won't resolve.

## Task 4 — Lab members / alumni / About / funding / resources

These are still plain HTML — edit the file directly:

- **Current members / alumni:** `people.html` (`block_lab` divs / the
  `alumni-list`).
- **About blurb, funding line, course notes:** the top of `index.html`.
- **Grants & awards:** `funding.html`. **Application/teaching resources:**
  `resources.html`.

When you touch the top banner or nav menu, remember it's **duplicated across all
four HTML files** — change them together so the site stays consistent.

Update the **"Last Updated"** date in the footer of the page(s) you changed.

## Deploying

```
git add -A
git commit -m "Update papers / photos / etc."
git push
```

Then give GitHub Pages a minute and hard-refresh <https://linnykos.github.io/>.
